First, the use of cases

  • 1. Follow the real-time price of stocks

    • 1.1 Copy the URL of the stock in the browser, paste the URL in the web's url blank;

    • 1.2 Select trigger condition is a digital comparison, when the page is loaded, click on the number of concern, the green box indicates that can be monitored, otherwise the red box indicates that can not be monitored;

    • 1.3 Select the comparison symbol and enter the desired number;

    • 1.4 Once saved and run, the app monitors the web page at a regular frequency, compares it to the target number, and will respond if the trigger condition is met.

  • 2. Follow the key information on the hot web sites such as Twitter

    • 2.1 Copy the hot spot URL in the browser, paste the URL in the web's url blank;

    • 2.2 Select trigger condition for text comparison - when keywords are displayed；

    • 2.3 Enter (multiple) keywords. For example, pay attention to social events, stars' name and other keywords;

    • 2.4 After you save and run, the app monitors the page every certain frequency, and if keywords appear on the page, the app will respond.

  • 3. Follow you for the latest web information

    • 3.1 Copy in the browser need real-time attention to the website (such as recruitment, company announcements, tourism, immigration and other official news), paste the web's url blank;

    • 3.2 Select trigger condition sit for text comparison - when any text changes;

    • 3.3 Note that you can wait for the entire page to load before it can be saved;

    • 3.4 After saving and running, the app monitors the page at regular frequency to monitor any text changes on the page, and if it is different from the previous lying page information, the app will respond.

  • Above, when the trigger condition is met, the app responds (the phone will ring, vibrate, flash, or local notifications).

Other issues

  • 1. What data on the web page cannot be monitored?

    • 1.1 Images, charts, icons, flash player controls, etc. in a web page cannot be monitored.

    • 1.2 It is not possible to monitor the data if it needs to be loaded by clicking or toggle buttons. Note that when the trigger condition is to monitor changes throughout the page, do not click or switch any buttons that need to load the data.

    • 1.3 If the user is required to log in, the page information loads data, it can not be monitored.

  • 2. When the phone is locked, the app will not be able to monitor.

    Because the iPhone is locked, the iPhone system pauses the running app. So do not manually lock the screen and turn on the never-lock screen, keep the front desk of the app and in normal operation. If in order to reduce the power consumption of the phone, you can turn on power-saving mode, the phone flips, the phone screen will be darkened.

  • 3. The app cannot be monitored when the app is in the background.

    Because the app is put in the background, the iPhone system pauses apps in the background in order to free up more memory. So put this app in the foreground and keep it up and running.
